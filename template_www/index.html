Visualizing Flip Graphs - by philippe ajoux and bobby prochnow

Source code: flip_graph fgraph geom triangulation or Fork on GitHub

Built with Processing

Instructions:

  1. Place points by clicking on in the applet. You may move points once they have been places.
  2. A triangulation of the points is created interactively as you add points.
  3. Once you have entered enough points (MAX of 8) click on the "Flip" button.
  4. This will generate the Flip Graph of the given embedding of points.
  5. You can click+drag to rotate the flip graph.
  6. Green nodes represent closer to the Delaunay Triangulation while red nodes are "poor" triangulations.

Purpose:

The purpose of this applet is to explore the flip graphs of small triangulations. Specifically, we felt it would be interesting to visualize the "utility" of each node in the flip graph. Here we define "utility" as the number of Locally Delaunay Edges at triangulation has. Thus, the Delaunay Triangulation will be the node with the highest utility. The flip graph lets you visualize how the utility changes as you flip away from the Delaunay Triangulation.

Algorithms:

  • Delaunay Triangulation - We implement an O(n^2) Delaunay Triangulation algorithm (we did not implement the History DAG) which is run interactively while the user places points. We modify it slightly so that we don't actually "flip-to-delaunay" so the triangulations we generate are not the actual Delaunay Triangulations. This is done on purpose since the flip graph will generate the actual Delaunay Triangulation.

  • Flip Graph Generation - A simple Breadth-First-Search is used to walk through all the possible edge flips in the triangulation. From this search we can generate the actual flip-graph.

  • Graph Drawing - A modified version of Tutte's algorithm is used to draw the flip graph. Since the flip graph may not be planar the graph we generate is not always a planar embedding. However, we tried to tweak the drawing algorithm to space out the nodes in a way that provided reasonable readability.

Getting the Code:

You can clone the project with Git by running: 

$ git clone git://github.com/pajoux/CompGeom10

** Note - The number of points is capped at 8 not because of our triangulation algorithm. This is mainly a cap emposed so that the flip graph generated remains reasonably legible. With more than 8 vertices, the flip graph can become rather large and unmanagable.
